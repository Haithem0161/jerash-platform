   IMPORTANT: This site serves both Arabic and English from the SAME URLs (language is client-side toggled, stored in localStorage). The correct hreflang approach for single-URL multilingual sites is:
   - Use hreflang="x-default" pointing to the canonical URL
   - Do NOT use separate /en/ and /ar/ hreflang tags (those URLs don't exist)
   - Search engines will see the same URL and use x-default as the fallback

   NOTE on TECH-03 ("connects Arabic and English versions"): For single-URL multilingual architectures like this site, x-default is the correct implementation. It tells search engines "this is the default/universal URL for this content regardless of language." The alternative (separate /ar/ and /en/ URLs with bidirectional hreflang) would require server-side rendering or URL-based language routing, which this SPA doesn't use. x-default satisfies TECH-03 by providing search engines with the canonical URL that serves both language versions.

   ```tsx
   // In SEO.tsx, add hreflang x-default:
   <link rel="alternate" hreflang="x-default" href={`${BASE_URL}${url}`} />

   // Update og:locale to use ar_IQ for Iraq market
   <meta property="og:locale" content={i18n.language === 'ar' ? 'ar_IQ' : 'en_US'} />
   <meta property="og:locale:alternate" content={i18n.language === 'ar' ? 'en_US' : 'ar_IQ'} />
   ```
